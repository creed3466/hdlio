## Infrastructure change — Docker image baking (P2-B.1)

First parallel rollout attempt (batch earlier on 2026-04-13, 14:43) failed on
`cc1plus` OOM during concurrent `catkin_make -j3` across three containers
(`mapOptmization.cpp` + GTSAM template instantiation saturated the 14 GB host).

Resolution path taken (Option A):
- Moved `catkin_make -j2 -DCMAKE_BUILD_TYPE=Release` from **runtime entrypoint**
  into **image build step** (`baselines/docker/lio_sam.Dockerfile`).
- A single build inside the image has full host RAM; `-j2` caps peak memory at
  `mapOptmization.cpp` — no OOM.
- Runtime containers now only execute: config overlay → roscore → roslaunch →
  `rosbag play` → odom extraction. Per-seq runtime ≈ 6 min (down from ≈ 15 min).

Files touched:
- `baselines/docker/lio_sam.Dockerfile` — bake LIO-SAM workspace + FLANN patch + launch asset
- `baselines/docker/lio_sam_ntu_run.launch` — minimal headless launch (module_loam + params.yaml)
- `baselines/scripts/run_inside_lio_sam.sh` — skip compile; only overlay/launch/record/extract
- `baselines/scripts/run_tier1_lio_sam_ntu.sh` — unchanged (concurrency=3, cpuset pools p1/p2/p3)
